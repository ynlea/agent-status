# UI 绿幕图标工作流

按“调研 → 清单 → 四图分组 → 逐组绘制 → 全部确认 → 抠图 → 校验”顺序执行。不要在图板尚未全部确认时开始抠图。

开始前读取 `references/workflow-reference.md`，使用其中的调研表、分组规则、提示词骨架和视觉检查项。需要创建分组清单时参考 `references/manifest.example.json`。

## 1. 调研真实 UI

读取用户指定的页面源码、组件、类型、状态、接口字段和当前图标入口。至少确认：

- 页面实际显示的字段、操作、状态和图例；
- 同一语义是否在摘要区、弹层、列表或设置区重复出现；
- 当前已有图标、装饰素材和回退图标；
- 图标在界面中的目标尺寸与背景色。

输出一份候选清单，标明“复用现有素材”或“需要新绘制”。不要只根据截图或经验猜字段。

## 2. 整理为每组 4 个

按界面区域和语义邻近关系分组，每组固定 4 个，顺序固定为左上、右上、左下、右下。

- 同一字段在多个位置出现时复用同一图标；
- 枚举档位优先复用一个字段图标，由 UI 改变大小、填充或颜色；
- 不为凑满 4 个随意发明无使用场景的图标；不足 4 个时继续检查相关状态、空态、提醒或设置入口，仍不足再向用户确认；
- 用户已给出精确清单时直接沿用；否则在第一次生图前展示分组并确认。

## 3. 建立风格基准

查看 3 至 6 张最有代表性的已有素材，记录轮廓、配色、材质、纹理、阴影、留白和小尺寸可辨识要求。把它们作为风格参考，不直接拼入新图。

绿幕主体不得使用接近 `#00FF00` 的亮绿色。叶片等绿色元素改用暖橄榄、棕绿或粉棕，避免抠图误删。

## 4. 一组一组绘制图板

优先使用用户指定的生图技能或接口；未指定时使用当前环境可用的位图生成技能。每次只生成一张图板，不并行批量生成。

每张图板必须满足：

- 正方形、严格 2x2 象限；
- 每象限一个完整、居中、互不相连的图标；
- 背景为均匀纯色 `#00FF00`；
- 无文字、数字、水印、边框、分隔线、投影、反射和白色贴纸描边；
- 四周与象限中线保留充分绿幕间距；
- 图标轮廓适合目标 UI 的小尺寸显示。

生成后立即查看图板，核对四个语义、象限顺序和禁用项。若失败，只针对一个明确问题重绘该组。全部图板确认后再进入抠图。

## 5. 分离目录和分组清单

保持生成图板、规范化图板、透明图标和校验报告彼此分离。推荐结构：

```text
<work-dir>/
├── boards-generated/
├── boards-normalized/
├── icons/
├── icon-groups.json
└── icon-qa.json
```

`icon-groups.json` 中每张图板必须准确列出 4 个最终图标文件名。文件名使用用户界面中的稳定中文语义或项目既有命名规范。

## 6. 抠图并校验

确认已安装 Pillow（Python 图像库）后运行：

```powershell
py -3 .agents/skills/create-ui-chroma-icons/scripts/extract_chroma_boards.py `
  --manifest <work-dir>/icon-groups.json `
  --boards-dir <work-dir>/boards-generated `
  --normalized-dir <work-dir>/boards-normalized `
  --icons-dir <work-dir>/icons `
  --qa-out <work-dir>/icon-qa.json
```

需要覆盖同名迭代产物时显式加 `--overwrite`。脚本会统一生成 `2048x2048` 图板和 `512x512` 透明图标，并检查：

- 每张图板恰好对应 4 个图标；
- 每个图标内容非空；
- 四角完全透明；
- 亮绿残留像素为 0；
- 文件名唯一且没有路径穿越。

## 7. 最终复核

脚本通过后仍需视觉检查：每组至少查看 1 张透明图标，重点检查细线断裂、绿边、主体被误删、多个对象粘连和小尺寸辨识度。

最终汇报图板数量、图标数量、保存目录、校验结果、生成技能或接口，以及实际使用的提示词骨架。不要把预览图、临时输出或凭据混入交付目录。
