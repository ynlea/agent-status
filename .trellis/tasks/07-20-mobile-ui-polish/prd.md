# 移动端导航与用量/设置页 UI 打磨

## Goal

修好会话详情返回路径，并打磨设备会话列表头、用量筛选与设置页对齐，使导航符合预期、关键信息可读、筛选区更紧凑干净。

## Background

- 会话详情路由挂在 Shell 外的顶层 `/sessions/...`，从设备会话列表再 `push` 进去后，系统手势返回会落到设备列表，而不是设备会话列表。
- 设备详情头区域固定高度 + 右侧插画占位，版本/最后心跳容易被截断。
- 用量页筛选占两行且「按模型/按设备」混在筛选里，实际只控制下方明细。
- 设置页连接配置行的 `>`、关于版本值视觉上未统一右对齐。

## Requirements

### R1 会话详情返回栈

- 从「设备 → 某设备会话列表 → 会话详情」进入时，系统手势返回与顶栏返回均回到**该设备会话列表**，不得直接回到设备总列表。
- 从首页会话卡片进入详情时，返回仍回到首页（既有路径不破坏）。

### R2 设备会话列表头信息完整可读

- 设备详情页顶部展示的监测端版本、最后心跳（及在线态/平台）不得因固定高度或插画遮挡被截断到看不全。
- 允许换行或加高头部区域；版本全文可读，最后心跳全文可读。

### R3 用量页筛选布局

- 日期、设备、渠道三个筛选同一行，整体更紧凑。
- 下拉触发器与展开菜单样式更干净（圆角、间距、文字层次、菜单背景与当前主题一致），避免当前「偏丑」的菜单观感。
- 「按模型 / 按设备 / 按渠道」从顶部筛选区移除；改为**明细标题行右侧**的小号分段切换（视觉参考设置页主题模式 `SegmentedButton`，但更小）。
- 切换仍只影响下方明细分组，不影响顶部汇总卡片的数据范围（日期/设备/渠道筛选逻辑不变）。

### R4 设置页对齐

- 连接设置三行（服务地址、访问密钥、后台监测）的右侧 `>` 同一竖线右对齐。
- 关于「版本信息」的版本号右对齐（与连接行值区对齐习惯一致）。

## Out of Scope

- 不改后端 API、用量统计语义、通知逻辑。
- 不改主题色体系与品牌组件大改版。
- 不新增筛选维度（模型列表筛选等）。
- Windows monitor BOM / Codex 目录问题不在本任务（已另处处理）。

## Acceptance Criteria

- [ ] 设备会话列表进入详情后，手势返回与按钮返回均回到该设备会话列表。
- [ ] 首页进入详情后，返回仍到首页。
- [ ] 设备详情头：版本与最后心跳在常见机型宽度下可完整阅读（允许换行）。
- [ ] 用量页：日期/设备/渠道同一行；明细标题右侧有小号分组切换；顶部不再有「明细」下拉。
- [ ] 用量下拉菜单在浅色/深色主题下可读、间距合理。
- [ ] 设置页连接三行 `>` 右对齐；版本信息值右对齐。
- [ ] `flutter analyze`（或项目既有静态检查）无新增 error；相关页面可编译。
