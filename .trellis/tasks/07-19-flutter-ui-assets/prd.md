# Flutter 原型图素材绘制

## Goal

依据 `source/原型图.png` 与「轻芽」只读监控产品路径，盘点 Flutter 客户端所需**自定义图片 / 插画 / 品牌化图标**，形成可绘制、可验收的完整素材清单，作为绿幕绘制 · 抠图 · 入库的唯一依据。

## User value

- 设计与实现共用同一份素材表，避免漏画、重复画、命名混乱
- 明确「必画插画 / 品牌 icon / 组件库可替代」，控制绘制范围与优先级
- 多姿态猫咪与设备场景补齐后，空态、错误、引导不再只有占位灰块

## 调研依据

- 原型图：`source/原型图.png`（7 屏）
  1. 首页（活跃任务）
  2. 设备页（设备列表）
  3. 设备详情（会话列表）
  4. 设置
  5. 首次使用引导
  6. 系统通知
  7. 空状态（暂无任务）
- 产品路径延展（原型未画全、实现会用到）：无设备、连接失败、需确认强调、工作中等待、离线设备、下拉刷新无结果等
- 现有仓库：无 Flutter 工程，无与原型一致的猫 IP / 设备插画 → **自定义位图一律「待绘制」**

## 产品约定（影响素材范围）

- 品牌名：轻芽
- 主 IP：同一只「芽芽」——布偶/重点色短毛猫 + 头顶小叶；**多姿态多表情，不换物种**
- 底栏：首页 / 设备 / 设置
- 主题：浅色暖底（原型）+ 跟随系统；**位图统一透明底，单套适配深浅色**
- Agent 标：默认 **自绘抽象**，不用官方 logo
