# 桌面灵动岛：取消主窗几何动画，稳定形态切换

## Goal

Windows 桌面端：主窗与灵动岛之间的切换改为**瞬时**（无 HWND 几何插值）；岛内四种形态（细条、状态胶囊、会话列表、通知播报）之间的切换**更稳、更顺**，消除偶发抽搐与尺寸跳变。

## Background

- 当前单窗方案：关主窗后主窗变形为屏顶岛；岛内用 `IslandPhase`（strip / hover / card）+ 播报状态组合出四种视觉形态。
- 主窗↔岛曾用 `_lerpBounds` 多步 `setBounds`，Windows 上与 Flutter 布局叠在一起，用户反馈割裂，**明确要求取消该动画**。
- 岛内形态已有 `AnimatedContainer` 与 HWND `resizeIsland`，但顺序/竞态不稳定（先缩窗再切 UI、固定 delay 收 HWND、并发 morph 互相打断），仍会出现抽搐。

## Scope

### In scope

1. **主窗 ↔ 岛：无几何动画**
   - 收起主窗进岛：直接进入岛 HWND 目标尺寸与岛 UI；不插值放大/缩小主窗。
   - 从岛/托盘展开主窗：直接恢复记忆的主窗尺寸与位置（及 chrome），隐藏岛 UI；不插值。
   - 允许极短的 show/focus 时序，但用户不可感知“窗在飞”。

2. **岛内四形态切换更流畅、稳定**
   - 细条（strip）
   - 状态胶囊（hover 摘要）
   - 会话列表（card 钉住列表）
   - 通知播报（announcement 宽胶囊）
   - 形态切换应可打断、可收敛到最终目标，不出现来回抽、中途尺寸跳一下再纠正。

### Out of scope

- 重新引入双窗 / 第二 Flutter Engine / 原生黑块岛窗。
- 改动会话业务逻辑、播报文案规则、过滤开关语义。
- macOS/Linux 桌面壳（产品目标仍为 Windows）。

## User-facing behavior

| 操作 | 期望 |
|------|------|
| 关主窗 / 最小化进岛 | 主窗立刻消失，屏顶出现岛（默认细条）；无窗体飞入飞出 |
| 托盘/岛打开主窗 | 岛立刻消失，主窗按记忆位置/尺寸出现；无放大动画 |
| 悬停细条 | 稳定展开为状态胶囊；离开后收条不抖 |
| 点击 | 稳定展开会话列表；再点/超时收成胶囊再收条 |
| 状态变更通知 | 稳定切到通知形态；结束后收胶囊/条，不与悬停/列表互相抽 |

## Constraints

- 继续单窗 + 岛形态 `DesktopWindowMode.island`。
- `window_manager` 在 Windows 上 `animate: true` 无效，不得依赖系统窗体动画。
- 岛 HWND 须贴内容尺寸（避免透明区挡点击），水平中心与顶边锚点保持现有约定。
- 改动集中在桌面壳：`window_controller`、`island_controller`、`island_bar`、相关常量。

## Acceptance Criteria

- [ ] 主窗进岛、岛回主窗路径中**不再**调用多步几何插值（如 `_lerpBounds`）；肉眼为瞬时切换。
- [ ] 关窗进岛后可见岛 UI；开主窗后岛 UI 不可见且主窗可正常交互。
- [ ] 细条 ↔ 胶囊 ↔ 列表 ↔ 通知 四条路径切换连续，连续快速悬停/点击/通知打断时**不出现明显抽搐或二次跳尺寸**。
- [ ] 收条后 HWND 与细条视觉对齐，无大块透明挡点击。
- [ ] 未改会话列表数据与播报文案语义；设置里「启用灵动岛」仍生效。
- [ ] Windows 桌面相关改动能通过现有静态检查/相关测试（若有）；无新增明显异常日志风暴。

## Notes

- 「四种形式」对应实现：strip、hover、card 列表、announcement（card + `hasAnnouncement`）。
- 用户明确：**不要**主窗↔岛动画；**要**岛内形态动画更稳。
