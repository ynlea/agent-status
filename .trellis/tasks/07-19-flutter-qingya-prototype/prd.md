# Flutter 轻芽 App 按原型完整重做

## Goal

用 **Flutter + 组件库**，按 `source/原型图.png` **完整重做**手机客户端「轻芽」：七屏信息架构、视觉气质、组件层级与文案与原型**对齐到可并排对比**（布局、圆角、色板、主副标题、状态色、底栏、空态、引导、系统通知样式）。素材缺口在实现中**随时补绘**，不凑合占位糊弄。

## User value

- 打开 App 即可认成原型里的「轻芽」，而不是旧 Android 单页原型堆砌
- 一眼看到：哪台机器、哪个目录、在做什么（短摘要 + 状态）
- 配置、列表、通知职责分离，日常只盯首页活跃任务

## 产品决策

1. **品牌**：轻芽；IP 用 `cat_*` 素材  
2. **底栏**：首页 / 设备 / 设置 + 设备详情  
3. **首页**：仅非 idle；排序 confirm > working > done；主标题=摘要，副标题=目录 · agent · 机器  
4. **设备 / 详情**：全机列表；会话活跃置顶，idle 可折叠  
5. **设置 + 首次引导**：URL、密钥、通知三开关、跟随系统；未配置全屏引导  
6. **通知**：标题=状态中文；正文=`机器 · agent · 目录 — 摘要`  
7. **只读**：无完整对话、无远程操控  
8. **视觉硬性**：**与原型并排对比一致**  
9. **栈**：Flutter；Theme 贴原型暖色控制台  

## 原型七屏映射

| 屏 | 关键点 |
|----|--------|
| S1 首页 | 头像+轻芽+铃；任务卡（色点/摘要/Agent 标）；空态趴睡猫 |
| S2 设备 | 机器名+在线点+箭头 |
| S3 详情 | 返回+机器名；会话卡同首页结构 |
| S4 设置 | 头像标题；地址/密钥/三色通知/主题 |
| S5 引导 | 大猫+欢迎+主按钮开始配置 |
| S6 通知 | app 标；确认红/工作橙/完成绿卡气质 |
| S7 空态 | 与首页 empty 共用 |

## Requirements

### R1 工程

- 新建 Flutter 工程（默认目录 `mobile/`）
- Android 为验收主平台；iOS 保持工程可开
- 接入 API：Bearer/密钥头、`/machines`、sessions、WS `notification`
- 本地存：URL、密钥、通知三开关

### R2 导航与数据

- 底栏三 Tab + 详情路由  
- 首页跨设备聚合非 idle  
- 连接态提示；失败可重试/去设置  
- `done` 停留跟服务端，App 不另做 10 分钟计时  

### R3 视觉还原（硬性）

- 暖米白底、白卡片大圆角、暖橙主色、状态红/橙/绿  
- 字体与间距层级对齐原型量感  
- **禁止**旧 Compose 单页充数  
- 资源：`source/ui-chroma/flutter_assets/` → 工程 `assets/images/`；缺图补绘入库  

### R4 通知

- Android 通知渠道 + 三开关  
- 关则不弹对应状态  

### R5 素材补齐

- 缺口表 → 绿幕补绘 → 同规范入库  
- 色点/色条/转圈代码绘制  

## Acceptance Criteria

- [ ] 七类界面可演示：首页有数据、首页空、设备、详情、设置、引导、通知样式  
- [ ] 与 `source/原型图.png` 并排截图：主色/卡片/底栏/头像/状态/文案结构无明显跑偏  
- [ ] mock 或真服：机器、会话、摘要、WS 能驱动 UI  
- [ ] 未配置 → 引导 → 填 URL/密钥 → 主界面  
- [ ] 通知三开关正确  
- [ ] 浅色贴原型；深色可读不崩  
- [ ] 无完整 prompt、无远程操控  
- [ ] 素材路径稳定，无临时路径依赖  

## Out of scope

- 重写监控/服务端业务（契约缺字段再单开）  
- 历史页、图表、远程批准  
- 商店 ASO 全套  
- 继续演进 Kotlin Compose 客户端（目录可暂留）
