# Flutter UI 原型对齐调研

## 结论

原实现已具备路由、数据仓库、基础主题和 40 枚透明素材，但只完成了通用 App 页面骨架。与原型的主要差距来自布局和信息层级，而不是素材数量。

## 改造前主要差距

| 页面 | 原实现 | 原型要求 |
| --- | --- | --- |
| 首页 | 品牌顶栏 + 普通任务列表 | 连接状态、猫咪主视觉、提示气泡、状态筛选、三层任务信息 |
| 设备 | 标题 + 通用白卡 | 品牌顶栏、设备计数、刷新、分色设备卡、设备图与猫咪状态图 |
| 详情 | 标准 AppBar + 平铺列表 | 设备状态主视觉、活跃/空闲分组、可折叠空闲会话 |
| 设置 | 可编辑表单与多个按钮 | 紧凑分组列表、连接摘要、通知总开关与三状态开关 |
| 首次引导 | 欢迎页和设置页分成两步 | 单页猫咪主视觉、地址/密钥输入、渐变主按钮 |
| 底栏 | 全宽直角底栏 | 页面内缩、全圆角、轻阴影、设备页蓝色选中态 |

## 素材复核

| 场景 | 处理 |
| --- | --- |
| 品牌头像、欢迎主视觉 | 复用 `cat_avatar`、`cat_welcome` |
| 任务三状态 | 复用 `cat_confirm`、`cat_working`、`cat_done` |
| 首页与设备空态 | 复用 `cat_empty_rest`、`cat_empty_devices` |
| 离线设备 | 复用 `cat_offline` |
| 设备类型 | 复用现有设备图，并在 UI 中增加深色遮罩保证小尺寸对比度 |
| 导航、设置、状态 | 复用现有透明图标；状态点、筛选片、卡片背景由 Flutter 绘制 |

首次实现复用了旧素材；用户视觉复核后确认旧素材风格仍与原型有明显差异。随后新增 `source/ui-chroma/v2/`：5 张 2×2 绿幕图板、20 枚透明 PNG，覆盖首页猫、四种列表猫、八种设备图和四个导航图标。气泡继续由 Flutter 代码绘制，并补上原型中的尖角尾巴。

用户再次复核后指出第二版猫咪被画成真实宠物照片，偏离原型的卡通布偶猫。第三版新增 `source/ui-chroma/v3/`：2 张 2×2 绿幕图板、8 枚透明 PNG。首页主视觉统一为同一只卡通布偶猫，列表猫保留颜色与品种差异，但统一使用动画电影式三维卡通体块；Flutter 页面已切换到第三版素材。
